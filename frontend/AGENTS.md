# Instructions for LLM Agents

This is a Next.js frontend for a concept-design application. It consumes the
typed SDK generated by the backend's `Requesting` concept. The API contract
lives at `../src/syncs/app.ts` and is imported via the `@backend/*` path alias.

## MCP Servers

This frontend expects the following MCP servers to be installed in the agent's
configuration (see `.cursor/mcp.json` for the project-level config):

| MCP Server | Command | Purpose |
|---|---|---|
| **shadcn** | `npx -y shadcn@latest mcp` | Add, search, and manage shadcn/ui components. |
| **next-devtools** | `npx -y next-devtools-mcp@latest` | Next.js build/test/lint assistant. |

Both MCP servers should be available before writing any frontend code.

## Typed Backend Integration

The frontend accesses the backend SDK and API contract types via path aliases:

```ts
// Typed client instantiation
import { createClient } from "@backend/sdk";
import type { AppApi } from "@backend/syncs/app";

export const api = createClient<AppApi>({ baseUrl });
```

`tsconfig.json` maps `@backend/*` to `../src/*`. When the backend adds or
changes endpoints (which define their routes in `src/syncs/`), the frontend
SDK types update automatically — no code generation step required.

- **Next.js 16** (App Router, RSC)

## Adding Components

Use the **shadcn MCP** to add shadcn/ui components. Do not hand-write component
code that the MCP can generate.

To add a component:
1. The shadcn MCP will read `components.json` for project configuration.
2. Components are installed into `src/components/ui/`.
3. Use the `npx shadcn add <component>` command if needed directly.

## Styling

- Global styles live in `src/app/globals.css` — CSS variables are defined there
  for light and dark themes.
- Use Tailwind utility classes with the `cn()` helper from `@/lib/utils`.
- Shadcn components use the theme tokens (`--primary`, `--background`, etc.)
  defined in globals.css.

## Running

```bash
bun install
bun run dev          # starts Next.js dev server (default :3000)
bun run build        # production build
bun run typecheck    # tsc --noEmit
```

Make sure the backend is running on the URL configured in
`NEXT_PUBLIC_API_BASE_URL` (default `http://localhost:8000/api`).

## Conventions

- Use `"use client"` directive on files that use hooks, browser APIs, or
  context providers (auth, theme, toasts).
- Keep server components (RSC) where possible — only opt into client
  components when needed.
- Follow the existing component pattern: one component per file, named export.
- Auth state is provided by `useAuth()` from `@/lib/auth`.
- Protected routes use `<RequireAuth>` from `@/components/app/require-auth`.
